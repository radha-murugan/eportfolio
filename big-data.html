Deciphering Big Data

Learning Outcomes

  • Master big data concepts for effective security risk management.
  • Develop critical analysis skills for proficiently addressing data wrangling challenges and selecting appropriate methodologies and tools.
  • Acquire comprehensive knowledge of diverse data types, formats, and storage requirements to make informed decisions in data management.
  • Learn efficient data exploration methods for presenting data effectively within organizational contexts.
  • Cultivate the ability to critically evaluate data readability, readiness, and longevity within the data pipeline while exploring cloud services and APIs for enhanced data interoperability and connectivity.

Unit 4 and 5: Data Cleaning, Data Transformation and Automating Data Collections

Learning Outcome:

This unit has assisted me in comprehending the basics of data cleaning and the process of writing, compiling, and utilizing Python scripts. Through the activities provided in this unit, I've gained a better understanding of identifying outliers, matching strings, and recognizing patterns.

The exercise focused on the UNICEF child labor dataset involved the utilization of a series of Python scripts for data cleaning. The activity commenced with the creation of a suitable header. Initially, data scraping techniques were employed to extract solely English headers, which were later augmented with additional data values and types. Subsequently, thorough examination of the data facilitated the removal of duplicate values, ensuring data uniqueness through the integration of Numpy libraries. This process underscored the importance of meticulous data design and the automation of processes. Despite encountering numerous challenges and errors during the automation of data cleaning, this exercise provided an invaluable opportunity to critically assess its limitations.

By the conclusion of this module, I have gained a comprehensive understanding of various essential aspects, including capturing raw data, the intricacies of data cleaning, integration techniques, principles of effective data design, and methods of visualization.

Unit 10: More on APIs (Application Programming Interfaces) for Data Parsing

Learning Outcome:

Through this module, I've gained a comprehensive understanding of the role of APIs in data parsing. I now appreciate how Big Data accesses various data sources beyond databases, including online platforms and social media, encompassing posts, surveys, and user information. Moreover, I've learned to configure data retrieval using different methods such as JSON, XML, and CSV, achieved through API calls with specific parameters and endpoints. Utilizing JSON parsing libraries, I've successfully parsed data received through APIs and even attempted to transform some of the parsed data as part of the API functionality. Furthermore, I've implemented error handling mechanisms to address parsing failures or unsuccessful transformations, maximizing the utility of handling APIs. As I delved deeper into the materials, I discovered the existence of industry-standard APIs utilized in various statistical analytical tools.

Indeed, I've also learned about the extensive capabilities of data parsing in APIs, including HTML parsing, regular expression parsing, PDF parsing, and even image parsing. It's fascinating to realize that APIs enable the slicing and dicing of data in any form, making it technically possible to manipulate and extract information from diverse sources.

Standard APIs prioritize security protocols to mitigate the risks associated with unauthorized access, data breaches, and service interruptions. Access controls are enforced through authentication, typically using OAuth, and authorization, employing Role-Based Access Control (RBAC). Transaction security is ensured during data transit by employing TLS certificates. Additionally, performance control mechanisms such as rate limiting and appropriate logging and monitoring are implemented. Given that these APIs are utilized in real-time, they offer enhanced efficiency and robust functionality.

In summary, I've gained a comprehensive understanding of how APIs are instrumental in data parsing. This involves accessing diverse data sources, retrieving data in various formats, and facilitating transformation for further processing. Additionally, I've learned about establishing a secure and reliable API environment, ensuring the integrity and confidentiality of the data being processed.



Unit 11

Learning Outcome:

This unit has provided me with a profound understanding of transaction processing. I was introduced to the ACID properties (Atomicity, Consistency, Isolation, and Durability) and their pivotal role in managing transactions and ensuring data integrity. Delving deeper, I explored DB transactions, including fully committed and non-committed transactions. I gained insight into scheduled transactions, logging mechanisms, monitoring system failures, and the significance of checkpoints within a database environment. Moreover, this unit elucidated the purpose of a transaction manager in overseeing day-to-day operations effectively.

I have gained a comprehensive understanding of ACID and its crucial role in ensuring data reliability and integrity. Implementing ACID principles safeguards against data inconsistencies caused by partially completed transactions, thus maintaining data integrity. While I haven't had the opportunity to test this functionality, I have acquired knowledge about recovery mechanisms that aid in restoring the database to a consistent state in case of failures. Additionally, I have learned about the utilization of data lakes in transaction management, enabling users to access a consistent view of their data even as it undergoes real-time updates.

Overall, this module has provided me with a deep understanding of how transaction management systems function and the critical considerations for recovery.
